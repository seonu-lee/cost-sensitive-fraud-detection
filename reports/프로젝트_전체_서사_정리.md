# 비용 민감형 금융 사기탐지 모델 개발 및 통계적 성능 검증
## Day 01~09 통합 서사

---

## 1. 프로젝트 개요

**목표**: 데이터 분석가/데이터사이언티스트 취업을 위한 실무 수준 포트폴리오 구축
**데이터셋**: IEEE-CIS Fraud Detection (Kaggle) — transaction 590,540건 + identity 144,233건, 사기율 3.5%
**차별점**: 통계학·경영학 복수전공 배경을 살려, 모델 성능 비교와 검증 전 과정에 통계적 근거를 부여

전체 흐름은 다음과 같다:

```
데이터 이해(01~03) → 모델 비교(04) → 통계적 검증(05) → 튜닝(06)
→ Calibration/Threshold(07) → 설명가능성(08) → 모니터링/Drift(09)
```

이 프로젝트가 다른 사기탐지 포트폴리오와 다른 지점은, 단순히 "높은 성능의 모델을 만들었다"에서 끝나지 않고 **그 모델을 실제로 믿고 운영할 수 있는가**를 통계적으로 검증하는 09단계까지 포함했다는 것, 그리고 그 과정에서 **분석 방법론 자체의 허점(데이터 누수)을 스스로 발견하고 규명했다**는 점이다.

## 2. Day별 진행 내용

### Day 1~2. 데이터 탐색 및 통계검정 (EDA)
- 기초 통계, 결측치/이상치 탐색, 변수별 분포 시각화
- 범주형 변수(가맹점 카테고리, 카드 타입 등) vs 사기 여부: 카이제곱 검정
- 연속형 변수(거래 금액 등) vs 사기 여부: t-test / Mann-Whitney U test
- 클래스 불균형(사기 3.5%) 정량화 및 이후 평가지표 선정에 반영

### Day 3. Feature Engineering
- **UID(가상 사용자 식별자)** 구성: card1, card2, addr1, addr2, D1(경과일) 조합으로 동일 사용자 추정 그룹화
- 사용자 단위 집계 변수(거래 빈도, 평균/표준편차 금액), 금액 편차(z-score), 결측 플래그 변수, 시간 파생 변수(hour/day/weekday/month) 설계
- 고카디널리티 변수(card1 등) PCA 압축 vs 그룹별 대표변수 두 버전 저장
- 메모리 다운캐스팅, 최종 208개 컬럼·결측치 0으로 정리
- **한계 명시**: UID는 "추정" 식별자이며 완벽한 사용자 매칭을 보장하지 않음

### Day 4. Cross-Validation 전략 및 모델 비교
- **GroupKFold(UID 기준)**로 동일 사용자 거래가 train/valid에 걸쳐 분리되지 않도록 설계 — 시간 순서와는 무관한 분할
- Logistic Regression, Random Forest, XGBoost, LightGBM 4개 모델 비교
- PR-AUC(핵심) + ROC-AUC(보조) 기준, 카테고리형 변수는 fold 내부에서 train fold의 타겟만으로 타겟 인코딩(누수 방지)
- 결과: Random Forest(PR-AUC 0.497) > LightGBM(0.486) > XGBoost(0.447) > Logistic Regression(0.337)
- Fold 4에서 전 모델 성능이 동시 하락하는 현상 발견 → UID 추정 방식의 구조적 한계로 해석
- XGBoost의 ROC-AUC가 LR보다 낮은 역전 현상 발견 → `scale_pos_weight` 과도 설정으로 추정, 튜닝 단계로 이월

### Day 5. 통계적 검증
- McNemar Test, DeLong Test, Bootstrap CI로 4개 모델 전체 쌍별(6개) 비교
- **RF와 LightGBM의 성능 차이가 통계적으로 유의하지 않음**을 확인
- 이 결과를 근거로 "실용적 이유(추론 속도, 배포 용이성, MLOps 확장성)로 LightGBM을 최종 후보로 선택"하는 의사결정 논리 확립

### Day 6. 하이퍼파라미터 튜닝
- RF, LightGBM, XGBoost 3개 모델을 Optuna로 튜닝 (LR 제외)
- "통계적으로 동급인 모델들 중 실용성과 튜닝 여력까지 고려해 최종 모델 확정"이라는 논리적 스토리 완성

### Day 7. Calibration & Threshold Optimization
- CalibratedClassifierCV로 예측 확률 보정
- 비용 민감형 임계값 최적화 (기획 단계부터 강조한 "비용 민감형" 정체성 반영)

### Day 8. Explainable AI (SHAP)
- 최종 Calibrated LightGBM 모델(`day8_calibrated_lgbm.pkl`)에 대해 SHAP 분석 수행
- 전역 중요도 상위: card1(1위), C13, C_PC1, card2, Transaction_day 등
- **(09에서 사후 발견)** card1 등 일부 변수의 SHAP 순위 해석에 데이터 누수 관련 유의사항이 추가됨 — 08 노트북에 참조 링크 표기

### Day 9. Drift 분석 — 프로젝트의 핵심 심화 구간

09는 단순히 "PSI 계산하고 끝"이 아니라, **분석 방법론 자체를 검증하고 스스로 오류를 발견·수정한 과정**이 서사의 중심이다.

**1) 시간 분할 설계**
- 04~07의 GroupKFold(UID 기준, 시간 무관)와는 별개로, month 0~4(학습) vs month 5+6(검증)으로 **시간 순서를 보존한 가상 분할**을 09 전용으로 새로 설계
- "모델 평가용 분할"과 "Drift 검증용 분할"의 목적이 다름을 명확히 구분해 문서화

**2) 입력 변수 Drift 탐지**
- 연속형 153개: PSI + KS Test 병행. Transaction_day(구조적 단조증가 변수)는 사전 제외
- 우선 점검 대상(PSI·KS 동시 높음): AllFlag_PC 계열(결측 패턴 주성분) — 광범위한 구조적 변화
- D11: PSI는 낮지만 KS는 높은 국소적 변화 — 특정 이산값(0, 43) 비율 변화가 원인, 동점 비중 66%
- 범주형 49개: card1 PSI=2.43이 나왔으나, top-N 재그룹핑 검증으로 **완전한 구조적 착시(희소 카테고리 왜곡)** 임을 실증적으로 확인 → PSI 0.003으로 소멸
- id_31, id_13은 재그룹핑 후에도 위험 등급 유지 → 실질적 Drift로 확정
- M1~M9(카드 일치 플래그): 저카디널리티라 왜곡 없이 신뢰 가능, Drift 확정

**3) 데이터 누수 자체 발견**
- 04~07(모델 학습/비교)에는 실제 GroupKFold를 사용했지만, **08 최종 배포 모델은 전체 데이터(590,540건)로 타겟 인코딩을 재계산해 재학습**된 것임을 확인
- 이 방식 자체는 배포 모델로서는 표준적이고 타당하나, **09처럼 시간 구간을 나눠 "미래 데이터에 대한 일반화"를 검증하려 할 때는 검증 구간의 정답 정보가 이미 인코딩값에 새어 들어간 상태**라는 근본적 모순을 실증적으로 검증 (특정 카테고리의 인코딩값이 학습구간 평균이 아닌 전체 데이터 평균과 일치함을 수치로 증명)

**4) 자체 보정 — 09 전용 재인코딩 보조 모델**
- 타겟 인코딩 대상 49개 변수를 학습 구간(month 0~4)만으로 스무딩(m=10) 재계산
- 스케일러도 학습 구간만으로 재fit, day8 최종 모델의 하이퍼파라미터를 그대로 재사용해 재학습
- Score PSI 재계산: 누수 있는 결과(0.0018) → 누수 제거 후 0.0319 (17배 상승, 여전히 "안정" 등급이나 방향성 확인)
- **평균 예측 사기확률이 학습(0.035) 대비 검증(0.018)에서 약 50% 하락** — PSI 지표만으로는 포착되지 않는 잠재적 성능 저하 신호를 별도로 발견

**5) SHAP 재계산 및 card1 재검증**
- TreeSHAP 계산이 98분 소요됨을 확인 → LightGBM 내장 `pred_contrib=True`로 대체(수 초~1분)
- Drift 위험 변수 14개 중 SHAP 상위 20위 안에 든 변수가 하나도 없음을 재현 확인(인코딩 방식과 무관하게 일관됨)
- card1은 재인코딩 후에도 SHAP 1위 유지(오히려 값 상승, 1.49→2.59) → 카테고리별 빈도-SHAP 관계 분석
- 스무딩 강도를 m=10→100으로 강화해 재검증: 51~500건 구간의 이상 급등(3.80→2.06)은 해소되었으나, 5,000건 초과 고빈도 구간의 SHAP(1.56→1.55)는 거의 변화 없음
- **결론: card1은 일부 과적합 요소가 있으나 실제로 카드 그룹별 사기율 차이를 반영하는 유의미한 예측 변수**로 최종 확정

## 3. 프로젝트 전체를 관통하는 핵심 메시지

1. **통계적 검증을 의사결정의 근거로 사용함**: "성능이 제일 높은 모델"이 아니라 "통계적으로 유의하게 더 나은 모델"을 최종 선택 기준으로 삼음(Day 5)
2. **지표를 맹신하지 않고 구조적 왜곡을 스스로 검증함**: PSI 절대값이 아니라 변수의 카디널리티·희소성까지 고려해 재검증(Day 9, card1)
3. **자신의 분석 과정에서 발생한 방법론적 오류를 스스로 발견하고 투명하게 수정함**: 08 최종 모델의 인코딩 방식이 09의 시간적 검증과 상충한다는 것을 실증 데이터로 증명하고, 별도의 보조 모델로 바로잡음
4. **단일 지표의 한계를 인지함**: Score PSI가 "안정"으로 나와도 평균값 이동(과소추정 경향)이라는 별도 신호를 놓치지 않고 짚어냄

이 네 가지가 "통계학 전공자다운 엄밀함"을 보여주는 핵심 차별화 포인트이며, 특히 4번째(자기 발견적 오류 수정)는 다른 포트폴리오와 가장 뚜렷하게 구분되는 지점이다.

## 4. 향후 과제 (미완료/확장 여지)

- card1 등 고카디널리티 변수의 재그룹핑 vs 스무딩 강화 간 최적 지점 탐색 (성능 손실 없이 과적합만 제거)
- 04~07의 GroupKFold fold별 모델을 이용한 out-of-fold SHAP 계산 (fold 모델 미저장으로 본 프로젝트에서는 미수행)
- 검증 구간 예측 확률 과소추정 현상의 원인 규명 (사기 패턴 자체의 시간적 변화 vs 모델의 시간 외삽 한계)
- 2단계 확장: Airflow/MLflow 기반 MLOps 파이프라인화 (기획 단계에서 논의됨)
